Motivation and goals of BioSimulations

More comprehensive and more predictive models have the potential to advance biology, bioengineering, and medicine. One of the most promising ways to build more comprehensive models is to combine multiple models of individual subsystems. For example, several researchers have are pioneering whole-cell models by combining multiple submodels that representing individual subcellular processes such as metabolism, transcription, and translation.

Building models by composition will likely require the collaborative effort of many computational and experimental biologists. Despite extensive efforts to develop standards such as COMBINE , SBML , and SED-ML and repositories such as BioModels, it remains difficult to combine models. One challenge to composing models is the diverse array of incompatible modeling formats and simulation tools. It is also difficult to find models to merge because the existing models are siloed in different repositories for different types of models. Furthermore, these repositories have limited capabilities for sharing associated resources such as training data, simulation experiments, and visualizations.

BioSimulations aims to help researchers combine models into more comprehensive models by making it easier for researchers to share and reuse models. First, BioSimulations provides authors a central portal for sharing models, simulations, and visualizations. Importantly, authors can share models in any format, authors can share simulations that require any simulator, and authors can share arbitrarily complex visualizations. Second, Biosimulations provides researchers a central portal for finding models and a simple web interface for reusing any model, simulation, or visualization. This is achieved through a collection of Docker images that abstract the details of each modeling framework, format, algorith, and simulation tool.

Downloading the models, simulations, and visualizations in BioSimulations

The models, simulations, and visualizations in BioSimulations can be programmatically obtained using our REST API . Documentation for the API is available at the same URL.

Obtaining the simulator Docker images

The simulator Docker images are available from DockerHub .

Obtaining the BioSimulations source code

The BioSimulations source code is composed of the following open-source packages:

  • Biosimulations : Core functionality
  • Biosimulations_utils : Utilities for encoding simulation experiments into SED-ML, encoding simulations studies in COMBINE/OMEX archives, and testing simulator images.
  • Biosimulations_COMBINE_archive_test_suite : Suite of COMBINE/OMEX archives for testing simulator images.

License

The models, simulations, and visualizations in BioSimulations are provided under the license specified for each resource. The simulator Docker images are provided under the open-source licenses documented for each image. The BioSimulations source code is provided under the MIT license . Contributing to BioSimulations

We welcome contributions to BioSimulations!

  • Models, simulations, and visualizations can be contributed through this website or our REST API. Please create an account to get started.
  • Simulator Docker images can be contributed by submitting GitHub issues . Please see the image template for information about the required format, a guide to building simulator images, and examples.
  • BioSimulations software: We welcome contributions by GitHub pull requests . Please see the Guide to Contributing for information about how to get started. Please also contact the developers to coordinate potential contributions.
