All of the simulation tools support the following features:

  • One step and timecourse simulations: sedml:OneStep and sedml:UniformTimeCourse.
  • Individual simulation tasks: sedml:Task.
  • Gata generators for time and individual variables: sedml:DataGenerator
  • Report outputs: sedml:Report.

Some of the simulation tools, such as tellurium, support the full SED-ML specification.

 Currently, reports are saved in Comma-Separated Values (CSV) format and plots are saved in Portable Document Format (PDF).

The columns of reports correspond to the simulation outputs specified in the corresponding SED-ML document (e.g., time, specific species). For time course simulations, the rows of reports correspond to the predicted time points. The first row of each report table describes the data represented by each column. Specifically, the first element of each column is the id of the simulation output (e.g., time or species id) represented by the column. Below is a sample report in CSV format and a formatted version of the same report.

Formatted sample report:

1.0 8.03.0 9.01.0 3.02.0
2.0 9.06.0 5.06.0 4.05.0
... ...... ...... ......

 A Docker image is a template for creating a Docker container. Typically, Docker images are containers for an operating system and additional programs. Docker images enable developers to encapsulate programs and their dependencies into a format which can be shared through online repositories such as Docker Hub and run on top of any operating system. A Docker container is a virtual machine. Typically Docker containers are created by instantiating Docker images.
