The BioSimulators standard for command-line interfaces for biosimulation tools is a set of requirements for the syntax and semantics of the inputs and outputs of biosimulation software tools. The standard ensures that simulation tools can be consistently executed with the same input arguments (paths to files that define models and simulations and paths to save simulation results) and that the simulation tools produce consistent outputs (reports and charts at consistent paths in consistent formats).

Simulation tools which implement the BioSimulators command-line interface standard should support the following command-line arguments:

  • -i, --archive: A path to a COMBINE archive which contains descriptions of one or more simulation tasks.
  • -o, --out-dir: A path to a directory where the outputs of the simulation tasks should be saved.
  • -h, --help: Optionally, this argument should instruct the command-line program to print help information about itself.
  • -v, --version: Optionally, this argument should instruct the command-line program to report its version and the versions of any critical dependencies.

Simulation tools which implement the BioSimulators command-line interface standard should also execute simulations and record their results according to the following conventions:

  • Execution of the contents of COMBINE/OMEX archives: Simulation tools should execute all simulations (e.g., all tasks in all SED-ML files) defined inside COMBINE/OMEX archives.
  • Formats of simulation results: Reports should be saved in Comma-Separate Values (CSV) format and plots should be saved in Portable Document Format (PDF) .
  • Encoding of simulation reports: Reports of simulation reports should adhere to the BioSimulators format for reports of simulation results.
  • File paths of simulation results: Reports should be saved to the same directory relative to the top-level output directory as the SED-ML file to the root of the archive. The file name of each output should be the id of the output plus the extension .csv for reports or .pdf for charts.

An an example, below is the documentation for the command-line interface for tellurium .

usage: tellurium [-h] [-d] [-q] -i ARCHIVE [-o OUT_DIR] [-v]

BioSimulators-compliant command-line interface to the tellurium simulation program .

optional arguments:
  -h, --help            show this help message and exit
  -d, --debug           full application debug mode
  -q, --quiet           suppress all console output
  -i ARCHIVE, --archive ARCHIVE
                        Path to OMEX file which contains one or more SED-ML-
                        encoded simulation experiments
  -o OUT_DIR, --out-dir OUT_DIR
                        Directory to save outputs
  -v, --version         show program's version number and exit
