HDF5
The BioSimulators format for SED reports of simulation results outlines how simulation results should be encoded into Hierarachical Data Format (HDF) 5 . These conventions are capable of capturing multidimensional reports, reports whose data sets have different shapes and data types, and reports whose data sets have repeated labels.

Reports of simulation results should be saved in HDF5 according to the following conventions:

  • Paths of reports: Within the HDF5 file, each report should be saved to a path equal to the combination of (a) the relative path of parent SED-ML document within the parent COMBINE/OMEX archive and (b) the id of the report. For example a report with id time_course_results in a SED-ML file located at ./path/to/experiment.sedml should be saved to the path path/to/experiment.sedml/time_course_results.

  • Data set shapes: The rows of each HDF5 dataset should correspond to the SED data sets (sedml:dataSet) specified in the SED-ML definition of the report (e.g., time symbol, specific model varibles).

    • Steady-state simulations: Tables should contain a single column of the predicted steady-state of each data set.
    • One step simulations: Tables should contain two columns of the predicted start and end points of each data set.
    • Time course simulations: Tables should contain number of points + 1 columns for each time point of the predicted time course.
    • Simulations of non-spatial models: Tables should not contain additional dimensions.
    • Simulations of spatial models: Tables should contain additional dimensions which represents the spatial axes of simulations.

    If the data sets of a SED report have different shapes, the SED data sets should be reshaped into a consistent shape by right-padding their values with NaN.

  • Metadata: The following metadata should be encoded into attributes of the corresponding HDF5 dataset.

    • Ids of SED data sets: The ids of the data sets should be encoded into the key dataSetIds. The value of this key should be an array of the ids of the data sets, in the order in which the data sets were defined in their parent SED document.
    • Labels of SED data sets: The labels of the data sets should be encoded into the key dataSetLabels. The value of this key should be an array of the labels of the data sets, in the order in which the data sets were defined in their parent SED document.
    • Data types of SED data sets: The data types of the data sets should be encoded into the key dataSetDataTypes. The value of this key should be an array of the data types of the data sets, in the order in which the data sets were defined in their parent SED document. The data type of each data set should be described using a NumPy dtype (e.g., int64) to indicate a data set whose value is non-null or __None__ to indicate a data set whose value is null.
    • Shapes of SED data sets: The shapes of the data sets should be encoded into the key dataSetShapes. The value of this key should be an array of comma-separated lists of the shapes of the data sets. The shapes of the data sets should be listed in the order in which the data sets were defined in their parent SED document.

Attributes of the HDF5 dataset for the SED report

  • dataSetIds: time, metabolite_a, metabolite_b, sum_metabolite_a_b, ratio_flux_c_d

  • dataSetDataTypes: float64, int64, int64, int64, float64
  • dataSetShapes: 14, 9, 11, 9, 14

Below are helpful tools for building reports of simulation results:

  • BioSimulators utils is a Python library which provides functions for generating reports to the above specifications.
  • h5py is a high-level Python library for reading and writing HDF5 files.
  • HDF5 libraries for C, C++, and Java.
