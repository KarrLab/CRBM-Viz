The BioSimulators format for reports of simulation results outlines how simulation results should be encoded into the rows and columns of tables, how the rows and columns should be labeled, and that tables of simulation results should be serialized to Comma-Separated Values (CSV) files.

Reports of simulation results should be saved in CSV format according to the following conventions:

  • Table layout: The columns of the reports should correspond to the data generators specified in the corresponding SED-ML document (e.g., time, specific species).
  • Row labels: Reports should not contain row labels.
  • Column labels: The first row of each report should contain column headings that indicate the id of the data generator represented by each column.

  • Results rows:

    • Steady-state and one-step simulations: Each report should contain a second row which contains the predicted value of each data generator.
    • Time course simulations: The subsequent rows of each report should correspond to the time points of the predicted time course. Each cell (row, column) should represent the predicted value of the corresponding data generator (column) at the corresponding time point (row).
